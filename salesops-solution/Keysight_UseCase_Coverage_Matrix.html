Keysight SalesOps — Use-case coverage matrix

Generated 2026-05-13 · audits every intent × gate × edge-case path the demo currently exercises.

This is the single-source reference the Keysight review team can use to confirm what the SalesOps automation does end-to-end, where the autonomy gates sit, and where every edge case lands. It is built from the live pipeline taxonomy, the four-gate confidence model, the AIOA webhook handoff, and the per-intent action matrix. Each row names the trigger, the gates the case passes through, the action ZBrain proposes, the downstream system that owns the work, and the failure / HITL path.

5. What ZBrain does NOT own

  • AIOA validation work — ZBrain triggers the webhook with the extracted PO context; AIOA executes the rule book and reports back. ZBrain never replicates AIOA's checks; the local mock simulates the round-trip for the demo.
  • Oracle EBS write — owned by AIOA today (when AIOA fires) and by Jitterbit for the non-AIOA flows once the middleware bridge is enabled from Settings → Integrations.
  • DocuNet filing — handed off to Keysight's document-classification overlay through Jitterbit. SharePoint is the active document store today; DocuNet is enabled from Settings → Integrations when the bridge is provisioned.
  • ServiceNow case lifecycle — owned by the Keysight dev team. ZBrain integration is read-only / write-on-handshake; we do not run Stage 4 substeps for ServiceNow.
  • Outbound SMTP send (under demo lock) — replies are drafted, attached, and logged as CommunicationLog rows; the actual SMTP transmission is suppressed during the demo to keep mailbox state stable.
